# 项目规则文档

## 1. 文档规则

### 1.1 Plan/Test 对齐规则

从本项目开始执行编码任务起，所有放入 `docs/plan/` 的实施计划文档，都必须在 `docs/test/` 下提供一个同编号、同主题的校验文档。

对齐要求如下：

- `docs/plan/1.md` 对应 `docs/test/1.md`
- 后续若新增 `docs/plan/2.md`，则必须同步新增 `docs/test/2.md`
- 不允许只写计划、不写校验

### 1.2 Test 文档职责

`docs/test/` 下的文档用于说明：

- 该阶段完成后应该如何验证
- 需要执行哪些命令或检查哪些结果
- 什么现象算通过，什么现象算失败

`test` 文档不负责描述实现方案，重点是验收口径。

### 1.3 Plan 文档职责

`docs/plan/` 下的文档用于说明：

- 目标是什么
- 要做哪些实现
- 目录、模块、脚本如何落地
- 当日或当阶段的边界与取舍

`plan` 文档不替代 `test` 文档，二者必须同时存在。

---

## 2. 执行规则

- 新增 `plan` 文档时，同一次提交中应同时补齐对应 `test` 文档
- 如果 `plan` 文档发生明显调整，对应 `test` 文档也必须同步更新
- 在验收某一阶段是否完成时，以对应 `test` 文档为准
