# 前端需求文档（极简 Vue 控制台）

本文档描述一个**极简**前端项目的目标与范围：仅作为本地后端 HTTP API 的消费方，不包含业务后端逻辑与模型调用。

---

## 1. 背景与目标

- **背景**：后端（`backend/`）已提供知识库索引、向量检索、`POST /api/chat` SSE 流式问答、`GET /api/status`、`POST /api/reindex` 等接口。
- **目标**：提供一个可本地运行的 Vue 界面，让用户能提问、查看流式回答与引用来源；可选查看知识库状态。
- **非目标**：多端适配、无障碍深度优化、商业化 UI、独立的用户体系与权限（均为后续扩展）。

---

## 2. 范围说明


| 纳入                                     | 不纳入（本阶段）               |
| -------------------------------------- | ---------------------- |
| Vue 单页应用，调用现有 REST/SSE                 | PDF 预览、文档编辑            |
| 对话区（提问 + SSE 增量展示答案）                   | 账号登录、OAuth             |
| 展示检索来源（`sources` 事件）                   | 复杂编排工作流                |
| （可选）状态页：`/api/status`                  | 服务端新增接口需求              |
| （可选）开发者操作：`POST /api/reindex`（可藏入二级入口） | 嵌入式 Qdrant/Ollama 管理面板 |


---

## 3. 建议技术选型

以下为建议默认值，可在实现阶段微调，但应保持「简单、易维护」：


| 项      | 建议                                                                                                                                            |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 运行时与构建 | Node 与包管理任选；脚手架 **Vite + Vue 3**                                                                                                              |
| 语言     | **TypeScript**                                                                                                                                |
| UI 体系  | **Material Design 3（M3）**：交互控件、表面层级、排版与状态遵循 M3；优先使用官方 **Material Web Components**（`@material/web`，自定义元素，可在 Vue 模板中直接使用）或与 M3 对齐的封装，避免混搭无关视觉体系 |
| 样式与布局  | **Tailwind CSS**（当前稳定版）：负责页面壳层、间距、Flex/Grid、响应式断点、辅助排版；**不**用 Tailwind 大面积覆盖 M3 组件内置主题色，以免与动态取色/容器 token 冲突                                   |
| HTTP   | `fetch`；SSE 使用 `ReadableStream` + `TextDecoder` 或 `**@microsoft/fetch-event-source`**（若遇跨域/Cookie 问题再引入）                                      |
| 路由     | 单页足够时可不用 `vue-router`；若拆分「对话 / 状态」两屏可加装                                                                                                       |


### 3.1 Material 3 与 Tailwind 的分工

- **Tailwind**：布局容器（如 `min-h-screen`、`flex`、`gap-*`、`max-w-*`）、外层留白与分区；可选用 `**@tailwindcss/forms`**、`**@tailwindcss/typography**` 仅作用于「自由文本/Markdown 预览区」（若有）。
- **Material 3**：按钮、文本字段、卡片/表面、列表、SnackBar（错误/成功提示）、对话框（若有）等；遵循 M3 的 **Elevation**、**Filled / Outlined**、**shape** 规范。
- **主题**：优先使用 Material Web 的主题/颜色 token（含浅色/深色切换能力时走 M3 约定）；Tailwind 的 `dark:` 可与全局 `class` 策略对齐，避免两套主题源互相打架。

## 4. 后端依赖与环境假设

- **Base URL**：通过环境变量配置（例如 `**VITE_API_BASE_URL=http://localhost:3000`**），前端所有请求前缀由此拼接。**禁止**在代码里写死后端端口（例外：开发环境默认值仅用于本地）。
- **跨域**：后端已内置 CORS（默认未设置 `CORS_ORIGINS` 时等价于允许 `*`，便于本地前端直连）。仍可用前端 **Vite `proxy`** 转发 `/api`，与直连二选一即可。
- **SSE**：`POST /api/chat` 返回 `text/event-stream`，前端需解析 `event:` / `data:` 行（见后端实现与 `docs/test/8.md`）。
- **健康检查**：可选用 `GET /healthz` 在做请求前探测，或在「关于 / 设置」区展示最近一次探测结果。

## 5. 功能需求

### 5.1 必做：对话页

1. **输入**：多行文本框 +「发送」按钮；发送前做基本校验（非空、trim）。
2. **请求**：`POST {base}/api/chat`，`Content-Type: application/json`，body：`{ "question": "<用户问题>" }`。
3. **响应**：按 SSE 解析事件：
  - `sources`：展示引用列表（至少含可读的来源路径或标题；具体字段以后端 payload 为准）。
  - `delta`：将 `text` **追加**到回答区域（流式）。
  - `done`：结束本轮流式输出，恢复按钮可点。
  - `error`：展示错误信息，结束本轮。
4. **异常**：网络错误、非 200、无法解析流时，给出可读提示，**不**假死界面。

### 5.2 可选：状态一览

- `GET {base}/api/status`，展示 JSON 中的核心字段即可，例如：`total_docs`、`total_chunks`、`failed_docs`、`indexing_docs`、`last_indexed_at`、`sources[]`（`name`、`path`、`doc_count`）。
- 用于确认后端与索引大致健康，无需图表。

### 5.3 可选：手动重索引

- `POST {base}/api/reindex`，body：`{}`（重试全部失败文档）或 `{ "doc_id": "<id>" }`。
- 建议放在「高级」折叠区，避免误触；成功后提示「已触发」并展示返回的 `triggered`（及若存在的 `failed`）。

## 7. 界面与交互（底线）

- **视觉**：整体为 **Material 3** 风格（表面层级清晰、控件状态明确）；页面骨架用 **Tailwind** 组织主次区域（如顶栏 / 主栏 / 侧栏来源列表）。
- **布局**：一屏内完成「问题输入 + 回答区 + 来源区」即可；移动端至少保证单列可用、触控目标符合 M3 最小点击区域。
- **状态**：发送中禁用重复提交或显示 **M3 进度指示**（线性或环形）；流式输出时允许「中止」（若实现 `AbortController` 取消 fetch 更佳）。
- **错误**：使用 **SnackBar** 或内联 **Banner**（M3）展示简短错误文案，避免仅剩浏览器 `alert`。

---

## 8. 交付与验收

1. 仓库内存在可独立安装、构建、启动的前端目录（例如 `frontend/` 或 `web/`，具体结构由实现 PR 决定）。
2. 后端按 README 启动后，前端通过配置的 `VITE_API_BASE_URL` 能完成一轮完整提问，并看到 **sources + 流式 delta**。
3. （若实现）状态页与重索引按钮行为与上述接口一致，无控制台未处理异常。
