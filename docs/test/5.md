# 第 5 天校验文档：Embedding 与入库主链路

## 1. 文档目的

本文档用于对齐 [第 5 天实施计划](../plan/5.md) 的验收方式，明确「Embedding、Qdrant 写入、SQLite 台账闭环」完成后应如何检查，以及何种现象视为可进入第 6 天检索开发。

本文档同时参考：

- [第 5 天实施计划](../plan/5.md)
- [代码编写计划](../coding-plan.md)
- [需求文档](../requirement.md)（§5.5、§7.1）
- [规则文档](../rule.md)

---

## 2. 校验范围

本次校验覆盖：

- Ollama **bge-m3**（或配置中的 `embedding_model`）Embedding 调用与重试
- Qdrant **向量 upsert**，Point ID 与 `chunk_id`（UUID）一致
- Qdrant **Payload** 字段与需求 §7.1 可对齐（至少涵盖计划要求字段）
- **更新**路径：旧 chunk 向量删除后再写入新向量
- **删除**路径：按台账 `chunk_ids` 删除向量并删除台账记录
- SQLite **台账** `status`、`chunk_ids`、`content_hash`、`error_msg`

本次不校验：

- 查询 Embedding、Top-K、Prompt、HTTP API（归属第 6～8 天）

---

## 3. 前置条件

- `backend/` 可执行 `bun run typecheck`
- 本地 **Ollama** 运行，且已存在配置中的 `**embedding_model`**（默认 `bge-m3`）
- **Qdrant** 可按现有 `[qdrant.ts](../../backend/src/store/qdrant.ts)` 方式访问（本机 REST 或容器）
- `backend/knowledge/`（或配置中的知识源）下至少有一篇内容足够长的 `.md`，经切块后能产生至少 1 个有效 chunk（满足 `min_chunk_length`）

至少：

```text
backend/src/ingest/embedder.ts

backend/src/store/vector-repository.ts
```

```bash
test -f backend/src/ingest/embedder.ts \
  && test -f backend/src/ingest/index-document.ts \
  && test -f backend/src/store/vector-repository.ts \
  && echo "day5 core files ok"
```

### 4.3 通过标准

- 上述文件存在
- `embedder` / `vector-repository` / `index-document` 职责分离；查询逻辑未误入 ingest

```bash
cd backend
bun run typecheck
```

- 退出码为 `0`

## 6. Embedding 行为校验

### 6.1 校验目标

- 对非空字符串可返回长度为 `embedding_dimensions` 的数组
- 失败时自动重试（最多 3 次），超过后调用方可将文档标为 `failed`

### 6.2 通过标准

- 故意断网或错误模型名称时，可观察到重试并最终失败路径（或集成测试模拟）

---

## 7. Qdrant 校验

### 7.1 校验目标

- 入库后可通过 **chunk UUID** 检索到对应 Point（或使用脚本 `retrieve`/`scroll` 验证存在）
- Payload 中含 `doc_id`、`source_path`、`chunk_text`、`section_title`、`indexed_at` 等关键字段（与实现约定一致）

### 7.2 通过标准

- 维度与 Collection 配置一致，无写入维度错误

---

## 8. 台账与增量一致性

### 8.1 校验目标

- 成功入库：`document_index` 中 `status` 为 `**done`**，`chunk_ids` 为 JSON 数组且与 Qdrant Point ID 集合一致
- Embedding 最终失败：`status` 为 `**failed**`，`error_msg` 非空
- **更新**：旧 UUID 对应的点在 Qdrant 中不存在（已删），新 UUID 存在
- **删除**：台账无记录；旧 UUID 点在 Qdrant 中不存在

### 8.2 通过标准

- 与 [需求文档](../requirement.md) §5.2 中新增/更新/删除描述一致（向量侧补齐）

---

## 9. 端到端校验

### 9.1 建议操作

运行项目提供的单篇入库脚本或单元流程（若计划中为 `index-single-doc` / 等价命令），对一篇 `readme-sample.md` 或等价样例执行完整入库。

### 9.2 通过标准

- 日志或控制台可看到：解析 → 切块 → Embedding → Qdrant → 台账

---

## 10. 完成定义

当以下全部满足时，可判定第 5 天校验通过：

1. `embedder.ts`、`vector-repository.ts`、`index-document.ts` 已落地并实现闭环。
2. Qdrant 与台账数据一致且可复查。
3. 失败路径可落到台账 `failed`。
4. `bun run typecheck` 通过。
5. 存在 [第 5 天实施计划](../plan/5.md) 与本文件的成对关系。

仍未满足时，不宜进入第 6 天检索模块开发。
